# Web実装・コーディングスキル

## 目的
`projects/` フォルダ内の指定された案件において、司令塔（Claude Cowork）が作成した要件定義（`requirements.md`）およびプロジェクトのコーディング規則に基づき、自律的に安全で高品質なコードの実装と修正を行う。

## 実行手順
1. 要件と規約の読み込み: プロジェクトフォルダ内の `requirements.md` と、リポジトリのルートにあるコーディング規則（例：`copilot-instructions.md` や `CLAUDE.md`）を読み込む。
2. 自律的な実装: `src/` フォルダ等にファイルを作成、または既存のコードを編集し実装を行う。
3. Artifactsでの提示: 実装結果やUIのモックアップは、ターミナルのログとして流すのではなく、人間が確認しやすいようにArtifacts（成果物ウィンドウ）を用いて視覚的に提示する。
4. 報告書の作成: 実装が完了後、プロジェクトフォルダの直下に `report.md` を作成（または上書き）し、実装機能の概要、遭遇した技術的課題、Claudeへ引き継ぐ事項を記録する。

## 動作の制約（セキュリティと堅牢性）
- **性善説の排除**: ブラウザ側のバリデーションやURLパラメータを信用せず、必ずサーバーサイドでのバリデーションと認可（他人のデータにアクセスできないかのチェック）を実装すること。
- **シークレットの保護**: APIキーやパスワードをソースコード内に直接記述しないこと。
- **異常系の考慮**: データベースの更新や外部APIの呼び出しを伴う処理は、途中でエラーが起きた場合のロールバック（トランザクション）と、二重送信防止（冪等性）を考慮して実装すること。
- **ライブラリの制限**: 存在が不確かなライブラリやマイナーなパッケージを推測で導入しないこと。新規パッケージの追加が必要な場合は、事前にユーザーに確認（Request Review）すること。
- **破壊的変更の禁止**: 既存のファイル構成やデータベーススキーマに対して、後戻りできない破壊的な変更を行わないこと。
