## 1. プロジェクト概要

### 目的

VRC（VRChat）向けのワールド制作およびアクセサリー・小物制作を主軸とする3DCGクリエイター兼デザイナー「Akuboxel」（運営: 大山 圭一朗）として、受注獲得と実績紹介を兼ねた1ページ完結のサイトを公開する。見込み顧客が「何を発注できるか」「価格感」「過去の作例」「どのチャネルで依頼できるか」を1画面で把握し、問い合わせフォームまたは外部プラットフォーム（ココナラ／Skeb／ENRAI）に到達する導線を構築する。

### 成功条件

- Lighthouse Performance スコアが モバイル 90 以上、デスクトップ 95 以上
- LCP が 2.5 秒未満、CLS が 0.1 未満、INP が 200ms 未満
- ファーストビューから問い合わせフォーム到達までのスクロール深度 75% 以上（GA4 スクロールイベントで計測）
- 問い合わせフォーム到達ユーザーのうち、送信完了率 15% 以上（GA4 イベントで計測）
- WCAG 2.1 レベル AA の主要要件（コントラスト比 4.5:1、キーボード操作、代替テキスト）を満たす
- 375px から 1440px の範囲で、横スクロールや要素崩れが発生しない
- 制作実績カードの画像読み込みは、遅延読み込みを適用し、初期表示に影響しない

### スコープ外（Non-Goals）

- EC 機能（サイト内でのデジタル商品の直接販売）。Booth / ENRAI 等の外部マーケットへのリンクに留める
- 会員登録・ログイン機能
- オンライン決済・見積もり自動生成
- 複数ページのブログ、コラム、記事CMS
- 多言語対応（初期は日本語のみ。英語版は Phase 2 以降）
- 動画の自社配信（YouTube 埋め込みで代替）
- A/B テスト基盤、MA ツール連携
- 依頼ステータスのトラッキング機能
- VRChat クリエイターエコノミー（プラットフォーム内課金）の案内。別途 Creator Economy Program Rules 同意が必要なため、Phase 2 以降で検討する

## 2. ターゲットと利用シーン

### 想定ユーザー

| 区分 | 属性 | 課題 | 本プロダクトで解決すること |
|------|------|------|------------------------|
| 主 | VRChat を日常的に利用するユーザー（20代〜30代）で、自分専用のワールドや小物を発注したい人 | 個人クリエイターに依頼したいが、制作範囲・価格・連絡手段が不明瞭で依頼を躊躇する | 対応業務・料金目安・作例・問い合わせ手段を1画面で提示し、5分以内に発注可否を判断できる |
| 副 | VTuber・配信者・バーチャルイベント主催者 | 配信用ワールドやブランディング小物を制作できるクリエイターを探している | 制作実績カードと SNS リンクで制作スタイルを可視化し、依頼前の技術判断を支援する |

### 主要ユースケース

1. X（Twitter）のポストから流入し、スマートフォンで制作実績を確認する
2. 料金メニューを閲覧し、ワールド制作またはアクセサリー制作の発注可否を判断する
3. 問い合わせフォームから依頼内容と予算感を送信する
4. 制作者の VRChat プロフィールや X アカウントを確認し、継続的にフォローする

#### ライセンス定義

- 個人ライセンス: 依頼主本人の個人利用・非商用利用に限定。VRChat 個人アカウントでの使用可
- 商用ライセンス: 依頼主本人が主催する有償イベント、投げ銭配信、収益化配信、法人利用での使用を許可
- 再販ライセンス: 依頼主が改変・二次配布・再販を行う権利を含む（Akuboxel のクレジット表記義務あり、原データの直接転売は不可）
- ビジュアル系（サムネ・ポスター）は性質上、再販ライセンスの対象外とする

#### 単品メニューの採算管理

基本料金を 3,000〜5,000円帯に収めるため、作業時間上限を越える要件は「超過分は追加料金」として明確に分離する。制作時間の実績を受注台帳に記録し、時給ベースで目標単価を下回る傾向が続くメニューは、次回改定で作業時間上限の見直しまたは価格見直しの対象とする。

### 割引プラン設計（F-04c で表示）

ワールド制作パッケージにおいて、Indie 区分のクライアントが以下のプランを選択すると、Indie Standard 価格から割引を適用する。割引は乗算方式で計算する（下限割を保つため）。

| プラン名 | 内容 | 割引率（対 Indie Standard 価格） |
|---------|------|----------------------------|
| 再販許諾プラン | 納品するワールドの汎用パーツ（家具、装飾モジュール、シェーダー、エフェクトセット）について、Akuboxel が再販・二次配布する権利を付与する。クライアント固有の造形物（キャラクター立像、固有IPモチーフ、サプライズ演出等）は再販対象外とする | -25% |
| 広告掲載プラン | 納品ワールドのスポーン近傍に、Akuboxel のクレジットプラーク（1m² 以下のサイン）を最低 6ヶ月間掲示する。掲載期間終了後は自動的に撤去可能。期間内の買い取り除去を希望する場合は +15% で対応 | -20% |
| 両方適用（推奨） | 上記両プランを同時に適用 | -40%（0.75 × 0.80 = 0.60、実質 -40%） |

#### 割引プランの運用上の注記

- 再販対象の切り分けは契約時に列挙方式で明示する。「どのアセットが再販可能か」を曖昧にしない
- 広告プラークのデザインは Akuboxel が提示し、世界観に馴染むバリエーションから選択可能とする
- 再販プランのみ適用、または広告プランのみ適用する場合は、上記の単独割引率（-25% または -20%）で Indie Standard 価格から計算する
- Corporate / Private 区分では原則として割引プランを適用しない（法人案件は排他性が前提）
- Indie Discount 適用後であっても、依頼主の事情変更により後から「排他への切り替え」を希望する場合は、当初の差額 +30% で再販許諾の解除に応じる

## 7. データモデル

### 主要エンティティ

本LPは静的コンテンツが中心のため、永続データモデルは持たない。フォーム送信時の一時データと、Works コンテンツの構造化データのみを扱う。

```
ContactSubmission（送信トランザクションのみ、DB保存はしない）
  - managementId: string (C{YYYY}-{XXXX} 形式、必須、サーバー採番)
  - name: string (1〜50文字、必須)
  - email: string (RFC 5322、必須)
  - category: enum (ワールド制作 / アクセサリー・小物 / ワールドギミック / その他、必須)
  - clientType: enum (Indie / Corporate・Private、必須)
  - discountPlan: enum (再販のみ / 広告のみ / 両方 / なし / 未定、必須、category=ワールド制作 のとき有効)
  - budget: enum (3万円未満 / 3〜10万円 / 10〜30万円 / 30万円以上 / 未定、必須)
  - deadline: string (自由記述、1〜100文字、任意)
  - urgency: enum (通常 / お急ぎ、必須)
  - message: string (1〜2000文字、必須)
  - submittedAt: ISO8601 タイムスタンプ
  - recaptchaScore: number (0.0〜1.0)
```

管理ID生成の実装方針（F-09b、PHP 実装例）

```php
// /api/contact.php 内
$submittedAt = (new DateTimeImmutable('now', new DateTimeZone('Asia/Tokyo')));
$year = $submittedAt->format('Y');
$seed = $submittedAt->format('Y-m-d H:i:s.u') . $email . ($_SERVER['HTTP_USER_AGENT'] ?? '');
$hash = strtoupper(substr(hash('sha256', $seed), 0, 4));
$managementId = "C{$year}-{$hash}";
```

- YYYY: 送信時刻（Asia/Tokyo）の西暦4桁
- hash: SHA-256 のうち先頭 16 ビット（4桁 hex）を抽出して大文字化
- 衝突耐性: 65,536 分の 1 未満。十分な識別性を確保しつつ DB 不要を維持
- 監査ログ: PHP `error_log()` または ConoHa Wing 標準のアクセスログに `submittedAt + managementId` を記録

```
Work（静的JSONまたはMDXで管理）
  - id: string (スラッグ、必須)
  - title: string (1〜60文字、必須)
  - category: enum (ワールド / アクセサリー、必須)
  - summary: string (2〜3行、必須)
  - imageUrl: string (WebP 推奨、必須)
  - tags: string[] (使用ツール・技術タグ、1〜5件)
  - externalUrl: string (Booth / VRChat ワールド等、任意)
  - createdAt: ISO8601 日付
```

### リレーション

なし。Works はフラットな配列で保持し、ソートは `createdAt` 降順とする。

## 9. セキュリティ要件・法令遵守要件

### 一般セキュリティ

- シークレット（SMTP パスワード、reCAPTCHA シークレット等）は GitHub Secrets と ConoHa 側の環境変数（`.env` ファイル、Web からアクセス不可な `public_html` 外に配置）で管理し、リポジトリへコミットしない
- `.htaccess` で `.env`、`.git`、`vendor/`、`node_modules/` 等の Web 公開を禁止する
- 入力検証はサーバーサイド（`/api/contact.php`）で型・長さ・許容文字種・必須項目を検証する。ホワイトリスト方式を採用する
- reCAPTCHA v3 のスコアが 0.5 未満の送信は受信拒否し、スコアが 0.3 未満の場合はサイレント破棄する
- エラー応答は本番環境で内部情報（スタックトレース、SQLクエリ、ファイルパス等）を返さない。`display_errors = Off` を厳守
- 個人情報保護法に基づく利用目的をプライバシーポリシーに明記する
- HTTPセキュリティヘッダ（CSP、X-Frame-Options、Referrer-Policy、X-Content-Type-Options、Strict-Transport-Security）を `.htaccess` の `Header set` で設定する
- HTTP は HTTPS へ 301 リダイレクトを `.htaccess` で強制する
- 外部リンク（Works、Channels、SNS）は `rel="noopener noreferrer"` を付与する
- フォーム CSRF 対策として、セッショントークンまたは Origin/Referer ヘッダ検証を実装する
- メール送信時のヘッダインジェクション対策として、改行コードを含む入力を拒否する

### 景品表示法に基づく価格・訴求表示

価格メニューと割引プランを掲示するため、景品表示法（不当景品類及び不当表示防止法）の要件を遵守する。詳細は `resources/2026-04-23-keihyoho-compliance.md` を参照。

価格表示の基本方針

- 「通常価格○○円が今なら△△円」形式の二重価格表示は使用しない。Akuboxel はローンチ時点で Indie Standard 価格の販売実績を持たないため、消費者庁ガイドラインの「最近相当期間にわたって販売されていた価格」要件を満たさない
- 割引表示は「権利付与の対価」として構成する。記載例: 「再販許諾と広告掲載を両方同意で Indie Standard 価格から -40% の Indie Discount 価格」。計算根拠（-25% × -20% の乗算 = 実質 -40%）と各プランの対価内容を近接配置で併記する
- 全価格表に「〜円〜（作業時間・仕様範囲の上限）」の前提条件を併記する
- 「最高」「業界一」「No.1」「大人気」「プロ品質」等の最上級表現・統計的訴求は使用しない。事実ベースの表現（「制作歴○年」「Blender・Unity を用いた一気通貫制作」）に限定する
- 「期間限定」「先着○名」「今だけ」等の限定表現は使用しない。割引プランは恒常的な権利付与型割引として運用する
- 顧客の声・納品実績の画像は、実在かつ同意取得済みのものに限る。LP の初期段階は空セクションとして設計し、実案件の同意取得後に追加する

景表法対応の画面要件（V-01 内で反映する項目）

- Pricing セクション（F-04）: 各価格の直下に「作業時間上限」「想定パーツ数」「前提とする仕様範囲」を注記
- Menu セクション（F-04b）: 同上。加えてライセンスプラン別の適用範囲を明記
- Discount Plan セクション（F-04c）: 権利付与の対価としての割引である旨と、各プランで Akuboxel が取得する具体的権利を明示
- 景表法準拠表示ページ（V-05、`/legal/keihyoho`）を新設し、詳細な価格前提条件と訴求根拠を記載する

社内記録の運用

- 受注台帳を運用し、案件ごとに「受注価格、適用区分（Indie Discount / Standard / Corporate）、割引プラン、実作業時間」を記録する
- 価格改定ログを運用し、「いつ、なぜ、どう変えたか」を残す
- 本記録は、景表法7条2項（不実証広告規制）に基づく合理的根拠資料として機能する

詳細は `.github/copilot-instructions.md`、`resources/2026-04-23-legal-tos-risks.md`、`resources/2026-04-23-keihyoho-compliance.md` を参照。
