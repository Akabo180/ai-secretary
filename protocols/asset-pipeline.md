# アセットパイプライン

3D・ゲーム・VR領域の案件で、シーンへアセットを組み込む前の検証と準備を定型化する。ここを飛ばすと、後工程で「シーンが壊れる」「ライセンス違反が判明する」「パフォーマンスランクが落ちる」などの手戻りが連鎖する。

## トリガー

- 「このアセットをシーンに入れて」「このモデルを使って」
- BOOTH/Asset Store/Sketchfab 等からアセットを入手した直後
- 新規 Unity/Blender/VRChat/cluster プロジェクトの Stage 2b 開始時

## 原則

- 出所不明のアセットは採用しない
- ライセンス確認を完了する前にリポジトリへコミットしない
- 最適化は「組み込み前」に行う。組み込み後の調整はコストが倍以上になる

## 実行手順

### 1. 出所とライセンスの確認

取得元とライセンスをアセット台帳 `projects/PRJ-XXX/resources/asset-ledger.md` に記録する。以下を必ず埋める。

| 項目 | 記入内容 |
|------|---------|
| アセット名 | ファイル名または作品名 |
| 取得元URL | BOOTH / Asset Store / Sketchfab / 自作 など |
| 作者 | 作者名またはブランド |
| ライセンス | 商用可否、再配布可否、改変可否、クレジット表記義務 |
| 価格・取得日 | YYYY-MM-DD、金額 |
| 用途 | このプロジェクトでの使用箇所 |

再配布不可のアセットは、リポジトリの公開範囲を private に固定する。public リポジトリで再配布不可アセットを扱う場合は `.gitignore` で除外し、別途配布経路を検討する。

### 2. 技術仕様の監査

モデル・テクスチャ・アニメーション・オーディオを、プラットフォームの制約に照らしてチェックする。

#### 3D モデル

| 項目 | 確認内容 | 目安 |
|------|---------|------|
| トライアングル数 | Blender/Unity の統計から測定 | 要件定義書の目標値以下 |
| マテリアルスロット数 | 1 メッシュあたりのマテリアル数 | 3 以下が望ましい |
| UV | 重複、UV0 の整備、ライトマップ用 UV2 の有無 | ベイクを行うなら UV2 必須 |
| リグ | ボーン数、Humanoid 互換、スケール（1.0） | VRChat ワールドの静的モデルには不要 |
| ピボット | 原点位置、初期回転 | シーン配置時の扱いやすさに直結 |
| ノーマル | スムージングの破綻、反転面 | 目視＋Blender の Select Non-Manifold |

#### テクスチャ

| 項目 | 確認内容 | 目安 |
|------|---------|------|
| 解像度 | 2 の累乗か | 512/1024/2048px を使い分け |
| 圧縮 | Unity の Import Settings | ASTC（Quest）、BC7（PCVR） |
| Mipmap | 有効化されているか | 通常シーンは有効 |
| カラースペース | sRGB / Linear | Normal Map は Linear |
| Alpha | アルファ不要のテクスチャに無駄なα値を持たせない | 該当チャンネルを削除 |

#### オーディオ

| 項目 | 確認内容 | 目安 |
|------|---------|------|
| フォーマット | wav/mp3/ogg | Unity は wav 推奨 |
| チャンネル | モノラル/ステレオ | 3D 音源はモノラル |
| ビットレート | 過剰な高音質は容量肥大 | 128kbps 前後 |
| Load Type | Decompress / Streaming | BGM は Streaming |

### 3. 最適化

監査結果に応じて、組み込み前に最適化を行う。

- LOD 生成（遠景用の低ポリ版）
- テクスチャアトラス化（ドローコール削減）
- 不要メッシュの削除（隠面、Cap ジオメトリの裏面など）
- マテリアルのマージ（同一シェーダー・同一テクスチャ）
- アニメーションのキーフレーム間引き

### 4. プロジェクトへの取り込み

- Unity の場合は `Assets/External/<ベンダー名>/<アセット名>/` 以下に配置する。ルート直下に置かない
- Blender の場合は `blend_assets/` 以下に配置する
- 取り込み直後に Import Settings を確認し、プラットフォーム別の圧縮設定を行う
- 大型バイナリ（100MB 超のテクスチャ、動画、音源）は Git LFS 対象であることを `.gitattributes` で確認する

### 5. アセット台帳の更新

取り込み完了後、`asset-ledger.md` に以下を追記する。

- 取り込み日
- 配置パス
- 最適化で行った変更（ポリ数削減、テクスチャ圧縮など）
- ベンダー同梱の元ファイルを保持する必要がある場合の保管場所

### 6. 受入判定

以下をすべて満たせばシーン組込に進む。

- ライセンスが用途に適合している
- 技術仕様が要件定義書のパフォーマンス予算以内
- アセット台帳に記録済み
- Import Settings が目標プラットフォームに適合

## 禁止事項

- ライセンス未確認のままシーンに配置する
- Asset Store 規約に反する形での再配布（プロジェクトの公開範囲を確認せずに push する等）
- 作者クレジット表記義務を無視する
- 画面に映らない部分のメッシュをそのまま残す（パフォーマンス予算の枯渇原因）

## 参考: よく使うライセンス種別

| ライセンス | 商用利用 | 改変 | クレジット | 再配布 |
|-----------|---------|------|-----------|--------|
| CC0 | 可 | 可 | 不要 | 可 |
| CC BY 4.0 | 可 | 可 | 必要 | 可（同条件） |
| CC BY-NC | 不可 | 可 | 必要 | 可（同条件） |
| BOOTH 通常販売 | 作者の規約次第 | 作者の規約次第 | 作者の規約次第 | 原則不可 |
| Unity Asset Store Standard | 可（プロジェクト同梱のみ） | 可 | 不要 | 不可 |

個別アセットのライセンスが上記と異なる場合は、作者の明示規約を優先する。
